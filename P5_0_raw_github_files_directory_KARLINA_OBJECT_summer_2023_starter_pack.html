For that reason, that directory web page will need to be saved multiple times in order to save each of the web pages which is referenced via exactly one uniquely corresponding hyperlink on that directory web page.

Note that the aforementioned GitHub repository (which is referenced on this web page using the green highlighted GitHub repository hyperlink in the yellow highlighted paragraph) can be saved in the WayBack Machine as a directory web page (using the “save outlinks” feature) such that the non raw files which are in that repository are saved as WayBack Machine web pages. To be clear, using the “save outlinks” feature of the WayBack Machine, inputting a public GitHub repository URL into the WayBack Machine web page save form and then clicking the SAVE button will save that GitHub repository as a web page in the WayBack Machine database and, also, save each non raw file which is in that GitHub repository as a web page in the WayBack Machine database (unless there is an exceptionally large number of files in that GitHub repository (which means that the GitHub repository will need to be saved multiple times using the “save outlinks” feature in order to create WayBack Machine saves of all of the non raw GitHub files which are in that repository)).
